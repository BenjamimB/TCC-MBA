# Plano de Testes — 3.1: Implementar gerenciamento de disponibilidade semanal

## Requisitos cobertos
- 1.1: Dias da semana ativáveis/desativáveis individualmente
- 1.2: Validação de horário de fim posterior ao de início
- 1.3: Cálculo automático de slots após salvar configuração
- 1.4: Propagação imediata das alterações (via evento AvailabilityUpdated)
- 1.5: Intervalo configurável entre consultas
- 1.6: Honrar slot já oferecido durante fluxo ativo (TTL Redis preservado)
