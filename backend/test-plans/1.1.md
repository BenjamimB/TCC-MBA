## Requisitos cobertos
- RNF-01: Disponibilidade 99,9% — health check é o mecanismo de monitoramento e sinalização de readiness

---

## 1. Funcionalidade
> Corretude dos comportamentos especificados nos requisitos.

Relevante: SIM

### Partições de equivalência identificadas
| ID | Partição | Tipo | Comportamento esperado |
|----|----------|------|------------------------|
| P1 | PostgreSQL UP + Redis UP | válida | GET /health → 200 `{ status: "ok", postgres: "up", redis: "up" }` |
| P2 | PostgreSQL DOWN, Redis UP | inválida | GET /health → 503 `{ status: "error", postgres: "down", redis: "up" }` |
| P3 | PostgreSQL UP, Redis DOWN | inválida | GET /health → 503 `{ status: "error", postgres: "up", redis: "down" }` |
| P4 | Ambos DOWN | edge | GET /health → 503 `{ status: "error", postgres: "down", redis: "down" }` |

TCP alvo: cobrir todas as partições acima.

### Casos de teste
| ID | Descrição | Partição | Tipo Jest |
|----|-----------|----------|-----------|
| TC-F-01 | GET /health retorna 200 quando PostgreSQL e Redis estão acessíveis | P1 | unit |
| TC-F-02 | GET /health retorna 503 quando PostgreSQL está indisponível | P2 | unit |
| TC-F-03 | GET /health retorna 503 quando Redis está indisponível | P3 | unit |
| TC-F-04 | GET /health retorna 503 quando ambos estão indisponíveis | P4 | unit |
| TC-F-05 | Corpo da resposta inclui campos status, postgres e redis em todos os cenários | P1-P4 | unit |
