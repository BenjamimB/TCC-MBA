## 1. Funcionalidade
> Corretude dos comportamentos especificados nos requisitos.

Relevante: SIM

### Partições de equivalência identificadas
| ID | Partição | Tipo | Comportamento esperado |
|----|----------|------|------------------------|
| P1 | Dia ativo com disponibilidade no intervalo solicitado | válida | Retorna lista de slots calculados |
| P2 | Dia inativo (`isActive=false`) | inválida | Retorna lista vazia |
| P3 | Intervalo sem disponibilidade configurada para o `dayOfWeek` | inválida | Retorna lista vazia |
| P4 | Slot com agendamento existente (booked) | inválida | Slot excluído da lista |
| P5 | Slot com lock Redis ativo | válida (marcado) | `isReserved=true` |
| P6 | Slot sem lock Redis | válida | `isReserved=false` |
| P7 | `startAt < now() + minAdvanceHours` | inválida | Slot excluído por antecedência insuficiente |
| P8 | `breakDurationMinutes > 0` | válida | Slots com intervalo entre eles |
| P9 | Intervalo com múltiplos dias | válida | Slots de cada dia com disponibilidade |
