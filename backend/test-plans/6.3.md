# Plano de Testes — 6.3: Implementar AITriageService

## Requisitos cobertos
- 4.1: Processar mensagem em linguagem natural e identificar intenção
- 4.2: Direcionar fluxo para processo correto após identificar intenção
- 4.3: Solicitar esclarecimento quando confiança insuficiente (< 0.7)
- 4.5: Responder em PT-BR cordial e profissional; fallback quando IA indisponível

---

## 1. Funcionalidade
> Corretude da detecção de intenção e geração de resposta.

Relevante: SIM

### Partições de equivalência identificadas
| ID | Partição | Tipo | Comportamento esperado |
|----|----------|------|------------------------|
| P1 | `detectIntent()` — resposta com confiança ≥ 0.7 | válida | retorna intent + confidence |
| P2 | `detectIntent()` — confiança < 0.7 | edge | retorna intent=unclear + confidence |
| P3 | `detectIntent()` — IA indisponível | edge | retorna AI_UNAVAILABLE |
| P4 | `detectIntent()` — resposta malformada (não JSON válido) | edge | retorna intent=unclear |
| P5 | `generateResponse()` — sucesso | válida | retorna texto em PT-BR |
| P6 | `generateResponse()` — IA indisponível | edge | retorna AI_UNAVAILABLE |
| P7 | `executeWithTools()` — tool_call retornado pelo provider | válida | retorna toolCalls + content |
| P8 | `executeWithTools()` — sem tool_calls (resposta direta) | válida | retorna content |

TCP alvo: cobrir todas as partições acima.
