# Plano de Testes — 7.3: Implementar WaitlistService com notificação FIFO

## Requisitos cobertos
- 9.1: Oferecer lista de espera quando não há slots disponíveis
- 9.2: Notificar automaticamente via WhatsApp após cancelamento (≤30s, FIFO)
- 9.3: Realizar agendamento automático quando paciente aceita a vaga
- 9.4: Liberar slot para agendamento geral se nenhum paciente aceitar no prazo
- 9.5: Pular paciente que já tem consulta confirmada no mesmo horário
- 9.6: Ignorar lista de espera se antecedência for inferior ao mínimo

---

> Corretude dos comportamentos especificados nos requisitos.

Relevante: SIM

### Partições de equivalência identificadas
| ID | Partição | Tipo | Comportamento esperado |
